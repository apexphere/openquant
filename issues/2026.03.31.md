# Trending-Up Detection Lag

## Problem

Trending-up labels consistently appear one step behind reality: they confirm after a rally is done and persist through the subsequent pullback.

## Evidence (detector-preview, default params, 2025-01-25 → 2026-03-29)

| Period | Label | Price Move | Problem |
|--------|-------|-----------|---------|
| Jul 8-14 | ranging-down | +10.1% (108k→119k) | Missed a massive rally |
| Jul 14 - Aug 4 | trending-up | -4.0% (119k→115k) | Confirmed AFTER rally ended, stuck during pullback |
| Aug 16-31 | trending-up | -7.8% (117k→108k) | Stuck bullish while price rolled over for 15 days |
| Oct 6-12 | trending-up | -7.8% (124k→114k) | Bullish at the market peak, price already crashing |

## Findings

### Finding 1: Confirmation counter oscillation deadlock (real bug, not the only cause)

The confirmation counter in `_apply_confirmation` tracks a single specific pending regime. When the raw signal oscillates between two exit regimes (e.g., trending-down and ranging-down), the counter resets every alternation:

```
confirmed = trending-up, price is falling:

Day 1: raw=trending-down  → pending=trending-down, count=1  (need 2 for bear_entry)
Day 2: raw=ranging-down   → pending=ranging-down,  count=1  (RESET! need 5 for bull_exit)
Day 3: raw=trending-down  → pending=trending-down, count=1  (RESET!)
Day 4: raw=ranging-down   → pending=ranging-down,  count=1  (RESET!)
...never exits trending-up
```

This is a real bug — the fix (group bearish/bullish signals in the same exit counter) was implemented and all unit tests pass. However, detector-preview output was identical for some periods because the raw signal doesn't always oscillate. The Aug 16-31 period IS caused by this bug (see Finding 3).

The fix was reverted to start fresh with the dominant regime redesign (below).

### Finding 2: Root cause is in `_classify`, not just confirmation

During the bad periods, `_classify` returns trending-up even while price is falling. This means one or more of:
1. **SuperTrend hasn't flipped** — price hasn't crossed below the SuperTrend line yet (factor=3.0 means the line is 3 ATRs away, wide band)
2. **ADX still shows strength** — a falling price can still have high ADX (strong move, just wrong direction from SuperTrend's perspective)
3. **CHOP still shows directional** — low choppiness during a steady decline
4. **SMA filter not catching it** — SMA(100) is slow, price may still be above it during early stages of a drop

## Design: Dominant Regime with Confidence (Refined Spec)

The fixed confirmation bars (bull_entry_bars=5, bear_entry_bars=2) are the wrong approach. They apply the same delay regardless of context. A breakout from ranging needs fast confirmation. A bounce during a crash needs heavy suppression. Same number can't serve both.

### Core concept

Maintain a **dominant regime** as a separate tracked state with a **confidence score** (0.0 to 1.0). The confidence is recalculated every bar from normalized indicator signals. It is not a counter that ticks up/down — it is a continuous score derived from the current indicator state plus a memory term that provides inertia.

This approach is inspired by Hidden Markov Model posterior probabilities (where a regime has a probability of being "active" at any point) but implemented as a deterministic weighted-signal composite — no statistical fitting, no training data, fully interpretable, and debuggable with `--debug`.

### 1. Confidence Calculation

#### 1a. Individual signal normalization (0.0 to 1.0)

Each indicator is normalized to a bullish confidence score between 0.0 (maximally bearish) and 1.0 (maximally bullish), with 0.5 meaning neutral/ambiguous.

| Signal | Bullish (toward 1.0) | Bearish (toward 0.0) | Normalization |
|--------|---------------------|----------------------|---------------|
| **SuperTrend direction** | Price > ST line | Price < ST line | Binary: 1.0 if bullish, 0.0 if bearish |
| **ADX strength** | ADX high (strong trend) | ADX low (no trend) | `clamp((adx - 15) / 30, 0, 1)` — maps ADX 15-45 to 0-1 |
| **CHOP directionality** | CHOP low (directional) | CHOP high (choppy) | `clamp((70 - chop) / 30, 0, 1)` — maps CHOP 40-70 to 1-0 (inverted) |
| **SMA position** | Price > SMA(100) | Price < SMA(100) | `clamp((close - sma) / (atr * 3), -1, 1) * 0.5 + 0.5` — distance-weighted, not binary |
| **MACD momentum** | MACD line > 0 | MACD line < 0 | `clamp(macd_line / (atr * 0.5), -1, 1) * 0.5 + 0.5` — scaled by ATR |

Why these normalizations:
- **ADX** has no theoretical max, but values above 45 are rare in crypto. The 15-45 range captures the meaningful variation. Clamping avoids outlier sensitivity.
- **CHOP** ranges 0-100 by definition; the 40-70 band is where crypto spends most of its time. Below 40 is strongly directional; above 70 is strongly choppy.
- **SMA** uses ATR-scaled distance rather than a binary above/below. Price 0.1% above SMA is very different from price 5% above SMA. The ATR scaling makes this volatility-adaptive.
- **MACD** is ATR-normalized to prevent absolute price levels from dominating (BTC at 120k has much larger raw MACD values than at 20k).

#### 1b. Composite bullish confidence (weighted sum)

```
bullish_confidence = (
    w_st   * st_signal     +
    w_adx  * adx_signal    +
    w_chop * chop_signal   +
    w_sma  * sma_signal    +
    w_macd * macd_signal
)
```

Recommended weights (must sum to 1.0):

| Signal | Weight | Rationale |
|--------|--------|-----------|
| SuperTrend | 0.35 | Primary direction signal, volatility-adaptive, fastest to flip |
| ADX | 0.20 | Trend strength — critical for distinguishing trending from ranging |
| CHOP | 0.15 | Choppiness filter — overlaps somewhat with ADX, lower weight |
| SMA | 0.20 | Structural bias — slow-moving anchor prevents false flips |
| MACD | 0.10 | Momentum confirmation — optional, lowest weight to avoid EMA lag contamination |

Why not equal weights: SuperTrend is the direction signal; the others are confirmation/strength signals. Equal weighting would let 4 weak confirmations override the primary direction signal. The 0.35 weight means SuperTrend alone cannot push confidence above 0.35, but combined with even one confirming signal it crosses 0.5.

Why not a voting system: Voting (majority wins) discards magnitude information. ADX at 21 vs ADX at 45 both count as "trending" in a vote. The weighted sum preserves how strongly each indicator signals.

#### 1c. Exponential memory (inertia)

Raw composite confidence is noisy bar-to-bar. Apply exponential smoothing:

```
smoothed_confidence = alpha * raw_confidence + (1 - alpha) * prev_smoothed_confidence
```

Where `alpha = 0.3` (configurable). This means:
- 30% of the score comes from the current bar's indicators
- 70% carries over from the previous bar's smoothed score
- Half-life is approximately 2 bars (`ln(0.5) / ln(0.7) = 1.94`)

This replaces the confirmation counter entirely. Instead of "wait N bars," the smoothing naturally requires sustained signal change before confidence moves meaningfully. A single contradicting bar moves confidence by at most `alpha * max_change = 0.3 * 1.0 = 0.3` — not enough to cross a tier boundary in one step from strong trending.

The alpha parameter is the primary knob for responsiveness vs. stability:
- `alpha = 0.5` — more responsive, faster transitions (good for short timeframes)
- `alpha = 0.2` — more stable, slower transitions (good for daily, the current use case)
- `alpha = 0.3` — recommended starting point for daily BTC

### 2. Direction and Regime Classification

The bullish_confidence score ranges from 0.0 to 1.0. Classification is symmetric around 0.5:

#### 2a. Direction determination

```
direction = "bull" if smoothed_confidence > 0.5 else "bear"
directional_strength = abs(smoothed_confidence - 0.5) * 2   # rescale to 0.0-1.0
```

`directional_strength` = 0.0 at the midpoint (no direction), 1.0 at the extremes (maximum conviction).

#### 2b. Regime tier thresholds (with hysteresis)

To prevent oscillation at tier boundaries, use asymmetric entry/exit thresholds:

| Tier | Entry threshold (directional_strength) | Exit threshold | Output regime |
|------|---------------------------------------|----------------|---------------|
| Strong trending | > 0.70 to enter | < 0.55 to exit | trending-up or trending-down |
| Weak trending | > 0.40 to enter | < 0.25 to exit | trending-up or trending-down (same label, different internal state) |
| Ranging | default state | -- | ranging-up or ranging-down |
| Chaotic | special condition (see section 4) | -- | chaotic |

The hysteresis gap (0.15 between entry and exit for each tier) prevents flickering. Once you enter "strong trending" at 0.70, you stay there until strength drops below 0.55 — a 0.15 buffer.

Why these specific numbers:
- **0.70 for strong trending**: Requires SuperTrend + at least 2 confirming indicators at high levels. With weights [0.35, 0.20, 0.15, 0.20, 0.10], getting to directional_strength=0.70 means bullish_confidence=0.85 — nearly all indicators agreeing.
- **0.40 for weak trending**: SuperTrend + one partial confirmer. Bullish_confidence=0.70 — the primary signal plus some support.
- **Below 0.40**: Indicators are mixed — this is ranging territory.

#### 2c. Output regime labels

The detector outputs the standard 5 labels that the composite strategy framework understands:

| Internal state | Output label |
|---|---|
| Strong bull | `trending-up` |
| Weak bull | `trending-up` |
| Strong bear | `trending-down` |
| Weak bear | `trending-down` |
| Ranging, direction=bull | `ranging-up` |
| Ranging, direction=bear | `ranging-down` |
| Chaotic | `chaotic` |

The internal strong/weak distinction affects transition speed (how quickly the detector leaves a trending state) but the output label is the same. Behaviors bound to `trending-up` do not need to know whether the trend is strong or weak — that is the detector's internal concern.

**Important**: The `trending-up` / `trending-down` labels are only emitted when directional_strength is above the weak-trending entry threshold (0.40). This is stricter than the current V5, which can emit `trending-up` when ADX is 10 and CHOP is 60 (as long as SuperTrend is bullish and the confirmation counter has filled). The new system requires genuine multi-indicator agreement before calling something a trend.

### 3. Dominance Decay Mechanics

Decay is not time-based. It is entirely indicator-driven through the smoothed confidence score.

#### 3a. How it works mechanically

Each bar:
1. Compute 5 normalized signals from current indicators
2. Compute raw bullish_confidence (weighted sum)
3. Apply exponential smoothing (`alpha * raw + (1-alpha) * prev`)
4. Compute directional_strength from smoothed confidence
5. Apply hysteresis thresholds to determine tier
6. Map tier + direction to output regime label

There is no separate "decay timer." When indicators start disagreeing, the raw confidence moves toward 0.5, which pulls the smoothed confidence toward 0.5 at rate alpha. This naturally produces the "Strong -> Weak -> Ranging" decay.

#### 3b. Decay speed depends on how wrong the indicators get

**Gradual deterioration** (ADX slowly fading, CHOP slowly rising):
- Raw confidence drifts from 0.9 toward 0.6 over several bars
- Smoothed confidence follows at 30% rate
- Takes ~5-7 bars to exit strong trending — appropriate for a slow fade

**Sudden contradiction** (SuperTrend flips):
- Raw confidence drops from 0.9 to ~0.35 in one bar (SuperTrend alone contributes 0.35 swing)
- Smoothed confidence drops from 0.9 to 0.9*0.7 + 0.35*0.3 = 0.735 in one bar
- Then 0.735*0.7 + 0.35*0.3 = 0.620 in two bars
- Exits strong trending (below 0.55 exit) in ~3-4 bars — appropriate for a sudden reversal

**Crash scenario** (SuperTrend flips + ADX surges in opposite direction + price below SMA):
- Raw confidence drops from 0.9 to ~0.15 in one bar
- Smoothed: 0.9*0.7 + 0.15*0.3 = 0.675 (bar 1), 0.675*0.7 + 0.15*0.3 = 0.517 (bar 2)
- Crosses 0.5 midpoint in 2-3 bars, direction flips to bear
- This matches the Oct 6-12 case where 2-bar response was appropriate

The smoothing naturally provides the asymmetric behavior we wanted: gradual deterioration is slow to exit, sudden crashes are fast.

### 4. Chaotic State Detection

Chaotic is distinct from ranging. Ranging means "no clear trend direction." Chaotic means "violent direction changes that make any signal unreliable."

#### 4a. Detection mechanism: direction flip frequency

Track the number of times the raw bullish_confidence crosses 0.5 (direction flips) in a rolling window:

```
flip_count = number of raw_confidence 0.5-crossings in the last N bars (N=10)
```

| Flip count (in 10 bars) | Interpretation |
|---|---|
| 0-1 | Stable — normal trending or ranging |
| 2-3 | Moderate — typical ranging behavior |
| 4+ | Chaotic — direction changing too fast to trust any signal |

Additionally, check for high volatility during the flips:

```
is_chaotic = (flip_count >= 4) AND (current_atr_percentile > 0.60)
```

The ATR percentile condition (ATR is in the top 40% of its recent 50-bar range) prevents labeling low-volatility chop as chaotic. Chaotic is specifically "high volatility + no consistent direction" — the political event scenario (tariff announcements, regulatory surprises).

#### 4b. Chaotic entry and exit

- **Entry**: When is_chaotic becomes True, override the regime to `chaotic` regardless of confidence level. This is a safety circuit-breaker.
- **Exit**: Require `flip_count < 2` AND `smoothed directional_strength > 0.30` for 3 consecutive bars. This is intentionally conservative — we want to be sure the chaos is over before re-engaging.

#### 4c. Chaotic as a regime label

`chaotic` becomes the 5th regime label, alongside trending-up, trending-down, ranging-up, ranging-down. It replaces `cold-start` in the behavior mapping (cold-start was always a proxy for "I don't know").

In the YAML config, strategies map it like any other regime:

```yaml
regimes:
  trending-up: trend_follow
  trending-down: trend_pullback_short
  ranging-up: bb_mean_reversion
  ranging-down: bb_mean_reversion
  chaotic: null          # flat — no trading
```

If a strategy does not define a `chaotic` mapping, it defaults to `null` (no behavior, no trading) — the safe default.

### 5. Edge Cases

#### 5a. Cold start (no history)

On the first bar with sufficient indicator data:
- Initialize smoothed_confidence to 0.5 (neutral)
- Initialize the regime to `ranging-up` (default safe state)
- The exponential smoothing will converge toward the true regime within ~5-7 bars (`1 - 0.7^7 = 92%` of the way to the true signal)
- No special cold-start label needed — ranging is the honest answer for "I don't know yet"

#### 5b. Gradual trend reversal (bull slowly becomes bear over months)

This is the design's strongest case. The confidence score will:
1. Start at ~0.85 (strong bull, all indicators agree)
2. ADX fades first as the trend exhausts (confidence drops to ~0.75)
3. CHOP rises as price action becomes choppy (confidence drops to ~0.65)
4. Price approaches SMA from above (confidence drops to ~0.55)
5. Exits strong trending, enters weak trending — regime is still `trending-up` but internally loosening
6. SuperTrend flips as price finally breaks below its trailing stop (confidence drops to ~0.35)
7. Direction flips to bear — regime becomes `ranging-down` or `trending-down`
8. If the bear side builds strength (ADX rises in new direction, price below SMA), confidence moves to strong bear

This gradual decay matches the reality of how trends end in crypto. There is no single "flip moment" — the system gracefully transitions through the tiers.

#### 5c. V-shaped recovery (sharp drop immediately followed by sharp rally)

Example: price drops 15% in 2 days, then recovers 15% in 2 days.

Day 0: Strong bull, confidence=0.85
Day 1: SuperTrend flips bear, raw drops to ~0.20. Smoothed: 0.85*0.7 + 0.20*0.3 = 0.655
Day 2: Price keeps falling, raw stays ~0.20. Smoothed: 0.655*0.7 + 0.20*0.3 = 0.519
Day 3: Price recovers sharply, SuperTrend flips bull, raw jumps to ~0.80. Smoothed: 0.519*0.7 + 0.80*0.3 = 0.603
Day 4: Recovery continues, raw ~0.80. Smoothed: 0.603*0.7 + 0.80*0.3 = 0.662

The regime would go: trending-up -> weak trending-up (day 1-2) -> ranging-up (day 2-3) -> ranging-up (day 3) -> weak trending-up (day 4). It never fully flipped to bear despite the sharp drop, which is correct behavior — V-shaped recoveries should not cause a full regime reversal.

If the flip_count during this period hits 4+ in 10 bars AND ATR is elevated, it could trigger chaotic instead — which is also correct (step aside during the volatility event).

#### 5d. Extended sideways market with occasional spikes

Confidence will hover around 0.5 (directional_strength near 0). ADX will be low, CHOP will be high. The regime will stay in ranging-up or ranging-down, occasionally flickering between them (which is fine — both typically map to the same behavior like BB mean reversion).

Occasional spikes will briefly push confidence toward one direction, but the smoothing prevents a single spike from triggering trending. The spike would need to persist for ~3-5 bars with confirming indicators to break through the 0.40 weak-trending threshold — which is the correct behavior.

#### 5e. Transition from ranging to trending (the Jul 8-14 problem)

This is the key case the new design must fix. In ranging:
- directional_strength is below 0.40
- Hysteresis exit threshold for ranging is N/A (it is the default state)
- The entry threshold for weak trending is 0.40

When a breakout happens from ranging:
- SuperTrend flips (0.35 weight shift)
- ADX may still be low (recently ranging)
- CHOP may still be moderate
- SMA may or may not confirm

With only SuperTrend confirming, raw confidence moves to ~0.65 (bullish side). One bar of smoothing: if previous was 0.45 (ranging), new is 0.45*0.7 + 0.65*0.3 = 0.51. Second bar: 0.51*0.7 + 0.70*0.3 = 0.567. directional_strength = (0.567-0.5)*2 = 0.134. Still below 0.40.

This is too slow for the breakout case. To handle this:

**Ranging-to-trending transition boost**: When the current tier is ranging AND the raw directional_strength exceeds 0.60 (strong raw signal), use `alpha_boost = 0.6` instead of the normal `alpha = 0.3`. This doubles the responsiveness for breakouts from ranging, while keeping the normal slow alpha for trend-to-trend transitions.

With boosted alpha: 0.45*0.4 + 0.65*0.6 = 0.57 (bar 1), 0.57*0.4 + 0.70*0.6 = 0.648 (bar 2). directional_strength = 0.296 (bar 2). By bar 3: ~0.70, directional_strength=0.40, enters weak trending. This is a 3-bar response for a ranging breakout — much better than the current 5-bar bull_entry_bars.

The boost only applies when transitioning OUT of ranging, not when in a trending state. This preserves the stickiness of established trends.

### 6. Implementation Notes

#### 6a. State tracked per bar

```python
# Internal state (not persisted between sessions, rebuilt from candle history)
self._smoothed_confidence: float = 0.5      # exponentially smoothed bullish confidence
self._current_tier: str = 'ranging'          # 'strong_trending', 'weak_trending', 'ranging', 'chaotic'
self._current_direction: str = 'bull'        # 'bull' or 'bear'
self._raw_confidence_history: deque          # last 10 raw values, for flip counting
self._confirmed_regime: str = 'ranging-up'   # output label
```

#### 6b. Configurable parameters

All thresholds should be constructor parameters with the recommended defaults:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `alpha` | 0.3 | Smoothing factor (0-1, higher = more responsive) |
| `alpha_boost` | 0.6 | Smoothing factor for ranging-to-trending breakouts |
| `strong_entry` | 0.70 | Directional strength to enter strong trending |
| `strong_exit` | 0.55 | Directional strength to exit strong trending |
| `weak_entry` | 0.40 | Directional strength to enter weak trending |
| `weak_exit` | 0.25 | Directional strength to exit weak trending |
| `chaos_flips` | 4 | Direction flips in 10 bars to trigger chaotic |
| `chaos_atr_pct` | 0.60 | ATR percentile threshold for chaotic |
| `w_st` | 0.35 | SuperTrend weight |
| `w_adx` | 0.20 | ADX weight |
| `w_chop` | 0.15 | CHOP weight |
| `w_sma` | 0.20 | SMA weight |
| `w_macd` | 0.10 | MACD weight |

#### 6c. Compatibility with detect_all

The `detect_all` method precomputes indicators once and loops over bars. The confidence calculation is per-bar (uses only current indicator values + previous smoothed confidence), so it fits naturally into the existing loop structure. No look-ahead.

Debug output should include: raw_confidence, smoothed_confidence, directional_strength, current_tier, and all 5 normalized signals. This makes every regime decision fully traceable.

#### 6d. Drop MACD initially

MACD adds complexity and its EMA-of-EMA nature was specifically called out as a problem in the V5 docstring. Recommendation: implement with 4 signals first (SuperTrend, ADX, CHOP, SMA) and adjust weights to sum to 1.0 without MACD:

| Signal | Weight (no MACD) |
|--------|-------------------|
| SuperTrend | 0.40 |
| ADX | 0.20 |
| CHOP | 0.15 |
| SMA | 0.25 |

Add MACD later if the 4-signal version underperforms on specific cases.

### 7. Trade-offs and Open Questions

1. **Alpha sensitivity**: The alpha parameter (0.3) is the single most important tuning knob. Too high and you lose stickiness. Too low and you miss breakouts. The ranging-to-trending boost (`alpha_boost`) mitigates this but adds complexity. An alternative: make alpha itself a function of the current tier (lower alpha in strong trending, higher in ranging). This is more elegant but harder to reason about.

2. **Weight sensitivity**: The weights are educated guesses. They should be validated against the known good/bad periods in the issue. If the detector is integrated into the optimizer, the weights themselves could be hyperparameters — but be cautious of overfitting (5 weights is a lot of degrees of freedom for regime detection).

3. **Chaotic as a distinct state vs. extreme ranging**: Chaotic could be mapped to an existing regime rather than adding a 5th label. The argument for a separate label: it carries distinct intent ("stop trading entirely"), while ranging still allows mean-reversion behaviors. The argument against: it adds complexity to every strategy's regime mapping. Recommendation: add it as a 5th label but default to null behavior when unmapped, so existing strategies work without modification.

4. **Hysteresis gap size (0.15)**: This was chosen to prevent oscillation at boundaries, but too-wide hysteresis creates dead zones where the detector is slow to react. The 0.15 gap means directional_strength must change by 0.15 to cross a boundary in either direction. With alpha=0.3 and a sustained signal change, this takes ~2-3 bars, which feels right for daily candles. For shorter timeframes, the gap might need to shrink.

5. **SMA period (100)**: The current V5 uses SMA(100). For the confidence-based system, SMA(100) on daily is ~3.5 months of data, which is quite slow. Consider SMA(50) for faster structural response, or keep SMA(100) for the anchor role. This is tunable and should be validated against the known periods.

6. **No SuperTrend factor change**: The current ST factor of 3.0 means the trailing stop is 3 ATRs away, creating wide bands. The confidence system helps with exit lag but does not fix the fundamental width of the SuperTrend band. If the Jul 8-14 breakout is still too slow even with the ranging-to-trending boost, consider lowering the ST factor to 2.5 for faster flips (at the cost of more false flips in ranging markets, which the confidence system can absorb).
